# 캠프 아이디어 보드

## 한 문장 정의

우리는 **AI 캠프 팀**의 프로젝트 주제 수렴 문제를 **90초 안에 설명 가능한 아이디어 카드와 랜덤 피치 뽑기**로 해결한다.

## 문제

팀 프로젝트 첫날에는 아이디어가 많지만, 문제·타깃·핵심 가치가 흩어져 저녁 시간에 주제를 정하지 못한다.

## 타깃

- 4일짜리 AI 빅데이터 캠프 참가 팀
- 5명 안팎으로 빠르게 MVP를 만들어야 하는 팀

## 핵심 기능

1. 아이디어 카드 등록: 제목, 해결할 문제, Wow 포인트를 입력하고 Supabase에 공유 저장한다.
2. 오늘의 피치 뽑기: 저장된 카드 중 하나를 무작위로 골라 발표 연습에 사용한다.

## 성공 기준

- 첫 화면에서 30초 안에 아이디어 카드 1개를 등록할 수 있다.
- 새로고침 후에도 등록한 카드가 유지된다.
- 버튼 한 번으로 발표할 아이디어가 선택된다.
- 공개 URL에서 등록한 아이디어가 새로고침 후에도 유지된다.
- Supabase 연결이 일시적으로 실패해도 현재 브라우저에서 아이디어를 임시 저장할 수 있다.

## Wow 포인트

회의 중 막힐 때 **오늘 발표할 아이디어를 한 번에 뽑아 바로 말할 수 있다.**

## 범위 밖

- 로그인·팀원 초대
- 실시간 협업
- OpenAI API 호출

## 기술 계획

- Day 1: 단일 `index.html`에서 Supabase 익명 인증·`ideas` 테이블 저장을 사용하고, 연결 실패 시 `localStorage`로 fallback한다. Vercel 또는 GitHub Pages에 배포한다.
- Day 2: 같은 URL에 Vercel Serverless Function과 OpenAI API 기반 에이전트 기능을 추가한다.
- Day 3: 평가·모니터링·가드레일을 붙인다.
