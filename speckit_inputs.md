# 캠프 아이디어 보드 - Spec Kit 입력 초안

`team_prd.md`를 기준으로 3차시의 PRD → 스펙 → 구현 흐름에 바로 붙여 넣을 수 있도록 정리한 입력 문서다.

## 1) `$speckit-constitution` 뒤에 붙일 원칙

- 첫 화면에서 30초 안에 아이디어 카드 1개를 등록할 수 있어야 한다.
- 핵심 사용자 흐름은 아이디어 등록과 오늘의 피치 뽑기 2개로 제한한다.
- 모바일 화면에서 한 손으로 입력·버튼 조작이 가능해야 한다.
- 외부 API나 로그인 없이도 데모 데이터로 기능 전체를 확인할 수 있어야 한다.
- 사용자에게 보이는 문구와 오류 안내는 한국어로 제공하고, 입력 내용은 안전하게 화면에 표시한다.

## 2) `$speckit-specify` 뒤에 붙일 한 줄 요약

AI 캠프 팀이 프로젝트 아이디어를 카드로 등록하고, 저장된 카드 중 하나를 무작위로 뽑아 90초 발표 주제를 정하도록 돕는 정적 웹앱을 만든다.

## 3) 유저 스토리와 이번 범위 밖

### 유저 스토리

1. 캠프 참가 팀원으로서 프로젝트 이름을 입력해 아이디어 카드를 등록하고 싶다.
2. 캠프 참가 팀원으로서 해결할 문제를 기록해 아이디어의 방향을 잊지 않고 싶다.
3. 캠프 참가 팀원으로서 Wow 포인트를 함께 적어 90초 발표에서 보여줄 장면을 정하고 싶다.
4. 캠프 참가 팀원으로서 새로고침해도 등록한 카드가 남아 있기를 원한다.
5. 캠프 참가 팀원으로서 버튼 한 번으로 오늘 발표할 아이디어를 고르고 싶다.

### 이번 범위 밖

- 로그인, 팀원 초대, 권한 관리
- 실시간 공동 편집과 외부 데이터베이스 연동
- OpenAI API를 이용한 아이디어 생성·평가

## 4) `$speckit-clarify`에서 물을 법한 질문과 권장 답

1. **데이터는 어디에 저장하는가?**  
   → Day 1은 가입·서버 없이 동작해야 하므로 브라우저 `localStorage`에 저장한다.
2. **카드가 하나도 없을 때 어떻게 보이는가?**  
   → 빈 상태 안내를 보여 주고 피치 버튼은 비활성화한다. 첫 실행에는 데모 카드 2개를 제공한다.
3. **무작위 선택 결과를 어떻게 확인하는가?**  
   → 같은 화면의 `TODAY'S PITCH` 영역에 선택된 제목과 Wow 포인트를 표시한다.

## 5) `$speckit-plan` 뒤에 붙일 기술 스택 한 줄

프레임워크 없이 단일 `index.html`의 내장 HTML/CSS/JavaScript와 `localStorage`를 사용하고, GitHub Pages 또는 Vercel로 정적 배포한다.
